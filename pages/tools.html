Ferramentas por contexto

Onde cada ferramenta costuma aparecer.

Esta organização ajuda a enxergar uso prático em vez de uma lista solta de nomes.

VS Code

Editor muito usado para escrever código, testar, organizar arquivos e desenvolver projetos.

Contexto: desenvolvimento, scripts, automação, projetos técnicos.

Git e GitHub

Ferramentas de versionamento e colaboração para acompanhar mudanças em projetos.

Contexto: desenvolvimento, documentação, portfólio, histórico de projeto.

Postman

Ajuda a testar APIs, entender respostas e validar integrações entre sistemas.

Contexto: desenvolvimento, integração, testes técnicos.

Excel

Ferramenta versátil para organizar dados, analisar informações e estruturar raciocínio.

Contexto: dados, governança, apoio analítico, organização de informação.

SQL

Linguagem usada para consultar, filtrar e manipular dados em bases relacionais.

Contexto: dados, BI, produto, operações, análise.

Power BI

Ferramenta de visualização e dashboards para apoiar leitura de cenário e decisão.

Contexto: BI, indicadores, gestão, analytics.

Jira

Ferramenta de acompanhamento de tarefas, backlog, fluxo e trabalho em time.

Contexto: agilidade, produto, projetos, times de tecnologia.

Trello

Ferramenta visual simples para organizar atividades, listas e andamento de trabalho.

Contexto: organização pessoal, times menores, fluxos simples.

Figma

Ferramenta usada para desenhar interfaces, protótipos e soluções visuais.

Contexto: UX, UI, produto, design de solução.

Miro

Quadro colaborativo usado para fluxos, jornadas, brainstorms e mapas visuais.

Contexto: UX, facilitação, workshops, agilidade, alinhamento.

Linux

Ambiente muito presente em servidores, infraestrutura, cloud e segurança.

Contexto: infraestrutura, cloud, segurança, operação técnica.

AWS / Azure

Plataformas de cloud usadas para hospedar, integrar e estruturar soluções técnicas.

Contexto: cloud, arquitetura, infraestrutura, soluções escaláveis.
Leitura útil

Você não precisa estudar todas as ferramentas desta página.

O importante é identificar quais fazem sentido para o caminho que você quer explorar primeiro.

O que costuma ajudar mais

  • Aprender ferramenta junto com um objetivo prático.
  • Relacionar ferramenta com área, rotina e tipo de problema.
  • Usar a ferramenta para construir algo pequeno desde cedo.

O que costuma atrapalhar

  • Tentar aprender muitas ferramentas sem saber por quê.
  • Escolher ferramenta por moda, e não por necessidade real.
  • Achar que dominar ferramenta sozinho resolve carreira.
Regra prática

Quando você entende o papel da ferramenta dentro da rotina, ela deixa de ser só nome e passa a virar apoio real.
