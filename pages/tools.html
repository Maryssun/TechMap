Ferramentas no dia a dia

Entenda que ferramentas aparecem em cada contexto da tecnologia.

Ferramenta não é carreira, mas ajuda a entender como o trabalho acontece na prática. Esta página organiza algumas das mais comuns para mostrar onde aparecem e para que costumam servir.

Como usar esta página

Use as ferramentas como leitura de contexto, não como lista para estudar tudo de uma vez.

Quem está começando costuma achar que precisa dominar muitas ferramentas antes de entrar na área. Na prática, isso quase nunca é verdade.

O mais importante é entender quais ferramentas aparecem no caminho que você quer explorar e aprender o suficiente para começar a usá-las em contexto real.

Desenvolvimento e construção

Ferramentas comuns para quem cria e testa soluções.

VS Code

Editor de código amplamente usado para escrever, organizar e testar projetos.

Contexto: desenvolvimento, automação, scripts, projetos técnicos.

Git

Sistema de versionamento usado para registrar mudanças e trabalhar com histórico de código.

Contexto: desenvolvimento, colaboração técnica, controle de mudanças.

GitHub

Plataforma usada para hospedar repositórios, colaborar e publicar projetos.

Contexto: portfólio, versionamento, times técnicos, documentação.

Postman

Ferramenta para testar APIs, entender respostas e validar integrações.

Contexto: desenvolvimento, testes, integração entre sistemas.
Dados e análise

Ferramentas que ajudam a ler, interpretar e visualizar informação.

Excel

Ferramenta versátil para organização, análise inicial e estruturação de informação.

Contexto: análise básica, organização de dados, apoio a leitura de cenário.

SQL

Linguagem usada para consultar, filtrar e trabalhar com dados em bancos relacionais.

Contexto: análise de dados, BI, produto, engenharia, operações.

Power BI

Ferramenta de dashboards e visualização para apoiar tomada de decisão.

Contexto: BI, gestão, analytics, indicadores.

Jupyter Notebook

Ambiente comum para análise exploratória, experimentação e modelos em Python.

Contexto: dados, IA, experimentação, prototipação analítica.
Organização e fluxo de trabalho

Ferramentas usadas para acompanhar trabalho, priorizar e organizar execução.

Jira

Ferramenta de gestão de tarefas, fluxo e acompanhamento de times.

Contexto: agilidade, projetos, produto, tecnologia.

Trello

Ferramenta visual de organização por quadros e cartões.

Contexto: organização pessoal, times pequenos, fluxos simples.

Azure DevOps

Plataforma que combina organização de trabalho, repositórios e entrega técnica.

Contexto: times de tecnologia, gestão de backlog, desenvolvimento.

ServiceNow

Ferramenta comum em gestão de serviços, governança e operação de TI.

Contexto: ITSM, governança, catálogo de serviços, processos.
Design, experiência e colaboração visual

Ferramentas que ajudam a pensar interface, experiência e construção colaborativa.

Figma

Ferramenta usada para desenhar interfaces, protótipos e soluções visuais.

Contexto: UX, UI, prototipação, produto.

FigJam

Espaço colaborativo visual para fluxos, brainstorms e organização de ideias.

Contexto: workshops, mapeamentos, ideação, descoberta.

Miro

Quadro colaborativo para estruturar fluxos, jornadas, processos e discussões visuais.

Contexto: agilidade, UX, facilitação, planejamento, alinhamento.

Notion

Ferramenta de organização de conteúdo, notas, documentação e estruturação pessoal ou de times.

Contexto: organização, documentação, acompanhamento, estudo.
Infraestrutura, ambiente e operação

Ferramentas e ambientes que aparecem em sustentação técnica e cloud.

Linux

Sistema muito presente em ambientes técnicos, servidores e operação.

Contexto: infraestrutura, cloud, segurança, operação técnica.

AWS

Plataforma de cloud com serviços para aplicações, dados, infraestrutura e automação.

Contexto: cloud, arquitetura, infraestrutura, soluções escaláveis.

Azure

Plataforma de cloud muito presente em contextos corporativos e integração com ecossistemas Microsoft.

Contexto: cloud, ambientes corporativos, infraestrutura, dados.

Docker

Ferramenta usada para empacotar aplicações e facilitar execução em diferentes ambientes.

Contexto: desenvolvimento, infraestrutura, ambientes, deploy.
Leitura importante

Ferramenta ajuda, mas direção vem do contexto.

Você não precisa estudar todas as ferramentas desta página. Precisa identificar quais aparecem no caminho que quer explorar e aprender em torno de uma prática real.

Quanto mais contexto você tem, mais sentido a ferramenta faz.
